Generated code

The model writes the job logic, not the platform.

Example: after repo A pushes to main, use a host bridge to update repo B documentation through a scoped GitHub grant.

Manifest boundary reviewable contract 
{
  "runtimeVersion": "2026-05-15",
  "id": "docs-syncer",
  "job": "After repo A changes on main, update repo B docs.",
  "trigger": {
    "type": "github",
    "events": ["push"],
    "repositories": ["WebMCP-org/repo-a"],
    "branches": ["main"]
  },
  "execution": {
    "mode": "think_extension",
    "extensionName": "docs_syncer",
    "toolName": "sync_docs"
  },
  "factsSchema": {
    "type": "object",
    "properties": {
      "lastSourceSha": { "type": "string" },
      "openProposalUrl": { "type": "string" }
    }
  },
  "githubPermissions": {
    "repoA": { "contents": "read" },
    "repoB": {
      "contents": "write",
      "pull_requests": "write"
    }
  },
  "endings": ["complete", "needs_human", "fail"]
}
Generated extension job-specific source 
({
  tools: {
    sync_docs: {
      description: "Update repo B docs after repo A changes on main.",
      parameters: {
        sourceSha: { type: "string" }
      },
      required: ["sourceSha"],
      execute: async ({ sourceSha }, host) => {
        const changed = await host.github.listChangedFiles({
          repo: "WebMCP-org/repo-a",
          ref: sourceSha
        });

        const docs = await host.github.readTree({
          repo: "WebMCP-org/repo-b",
          path: "docs/sdk"
        });

        const patch = await host.github.proposeDocsPatch({
          targetRepo: "WebMCP-org/repo-b",
          changed,
          docs,
          skill: "repo-docs-sync"
        });

        if (patch.kind === "empty") {
          return host.endings.complete({ summary: "Docs already current." });
        }

        await host.facts.patch({
          lastSourceSha: sourceSha,
          openProposalUrl: patch.pullRequestUrl
        });

        return host.endings.complete({
          summary: "Opened docs update PR.",
          changeProposalUrl: patch.pullRequestUrl
        });
      }
    }
  }
})

Rules

The product rules stay small.

1

One job

One or two sentences. Anything larger becomes multiple Nanites.

2

One harness

A host Think run loads generated extension, script, or facet code.

3

Trigger

PR, push, cron, webhook, or manual action.

4

Ending

complete, fail, or needs_human. No ambiguous terminal state.

5

Least privilege

Ask humans for narrow grants. Log every grant and tool action.
